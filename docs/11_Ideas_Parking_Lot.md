# Purpose

This document serves as a repository for ideas, experiments, feature requests, and long-term product improvements that are **not part of the current project scope**.

The purpose of this document is to encourage creativity without introducing scope creep into the current release.

**Important Rule**

> **An idea in this document is not considered part of the product until it is formally added to the Product Requirements Document (PRD).**

---

# Evaluation Criteria

Before moving an idea into the PRD, it should answer the following questions:

* Does it solve a real user problem?
* Does it align with the Project Vision?
* Does it improve the learning experience?
* Is it technically feasible?
* Does it fit within the planned release?
